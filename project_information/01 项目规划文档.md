# AI Coding Agent Framework - 项目规划文档

## 项目概述

**项目名称**: OpenSkill Agent Framework (开代理技能框架)

**项目目标**: 构建一个轻量级、开源、易于部署的 AI Agent 框架，支持：
- 多轮对话能力
- 自动技能发现和执行
- Tool Call 支持
- 完整的执行链路追踪
- Web UI 交互

**核心特点**:
- ✅ 轻量级：基于 Python，依赖最少化
- ✅ 开源：完整代码，易于定制和扩展
- ✅ OpenAI Compatible：支持任何 OpenAI-compatible 的 LLM
- ✅ Skill 自动化：自动发现、加载、执行技能
- ✅ 可视化：完整的执行链路展示
- ✅ 无厂商锁定：不依赖特定的闭源工具

## 技术栈

### 核心依赖
- **Python**: 3.9+
- **LangChain**: 用于 DeepAgent 集成

- **SQLite/JSON**: 执行历史和状态存储

- **Poetry**: 依赖管理
- **pytest**: 单元测试
- **black/flake8**: 代码格式化
- **mypy**: 类型检查

## 核心概念

### 1. Skill（技能）
技能是一个独立的、可复用的模块，包含：
- **定义文件** (`skill.json`): 描述技能的元数据
- **配置文件** (`config.yaml`): 技能的配置
- **实现文件** (`tools.py`): 技能的具体实现
- **文档** (`README.md`): 使用说明

**Skill 的结构**:
```json
{
  "name": "web_search",
  "version": "1.0.0",
  "description": "搜索网络信息",
  "category": "information_retrieval",
  "tools": [
    {
      "name": "search",
      "description": "搜索网络",
      "params": {
        "type": "object",
        "properties": {
          "query": {"type": "string"}
        }
      }
    }
  ],
  "dependencies": [],
  "author": "user",
  "license": "MIT"
}
```

### 2. Tool（工具）
工具是 Skill 中可被 LLM 调用的具体函数。每个 Tool 需要：
- 清晰的名称和描述
- JSON Schema 定义参数
- 具体的 Python 实现
- 错误处理和返回值

### 3. Execution Chain（执行链路）
完整的执行流程，包括：
1. 用户输入解析
2. LLM 推理和规划
3. 工具调用和执行
4. 结果反馈和迭代
5. 最终响应生成

## 非功能需求

### 性能
- **消息延迟**: < 1s（不包括 LLM 推理时间）
- **Skill 加载**: < 500ms
- **Tool Call 执行**: 支持并发执行
- **内存占用**: < 500MB 基础运行

### 可靠性
- **错误恢复**: 自动重试失败的工具调用
- **状态持久化**: 支持恢复中断的执行
- **Fallback 机制**: 支持多个 LLM 服务商

### 可维护性
- **代码覆盖率**: > 80%
- **类型注解**: 100% 的代码覆盖
- **文档**: 每个公共 API 都有文档
- **日志**: 支持多级别日志和追踪

### 用户体验
- **易部署**: 支持 Docker、Docker Compose
- **易扩展**: 清晰的 Skill 开发指南
- **易使用**: 直观的 Web UI，完整的示例

## 关键风险和应对

| 风险 | 影响 | 应对措施 |
|------|------|--------|
| LLM 调用失败 | 功能不可用 | 实现重试、超时和 fallback 机制 |
| Skill 加载慢 | 用户体验差 | 实现缓存和异步加载 |
| Tool Call 解析错误 | 执行中断 | 严格的 schema 验证和错误恢复 |
| 并发问题 | 状态不一致 | 使用锁和事务来保护共享状态 |
| 安全漏洞 | 数据泄露或代码执行 | 沙盒执行、权限控制、输入验证 |
